    示例

    以坐标值方式定位

    浮层可以在页面内按照自定义方式定位。

    居中浮层

    点击按钮，会出现视口范围内居中的浮层。

    <template>
<article>
  <veui-button
    @click="centerVisible = !centerVisible"
  >{{ centerVisible ? '关闭' : '打开' }}居中浮层</veui-button>
  <veui-overlay
    :open.sync="centerVisible"
    overlay-class="center-overlay"
  >居中浮层</veui-overlay>
</article>
</template>

<script>
import { Overlay, Button } from 'veui'

export default {
  components: {
    'veui-overlay': Overlay,
    'veui-button': Button
  },
  data () {
    return {
      centerVisible: false
    }
  }
}
</script>

<style lang="less">
.center-overlay {
  position: fixed;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
  width: 200px;
  height: 100px;
}
</style>

    相对已有元素定位

    浮层可以相对于某个已存在的 DOM 节点定位。

    相对定位浮层

    点击按钮，会出现一个紧贴按钮显示的浮层。

    <template>
<article>
  <veui-button
    ref="relative"
    @click="relativeVisible = !relativeVisible"
  >{{ relativeVisible ? '关闭' : '打开' }}浮层</veui-button>
  <veui-overlay
    target="relative"
    :options="{
      attachment: 'bottom left',
      targetAttachment: 'top left'
    }"
    :open.sync="relativeVisible"
    overlay-class="relative-overlay">相对定位浮层</veui-overlay>
</article>
</template>

<script>
import { Overlay, Button } from 'veui'

export default {
  components: {
    'veui-overlay': Overlay,
    'veui-button': Button
  },
  data () {
    return {
      relativeVisible: false
    }
  }
}
</script>

    层叠顺序管理

    越靠后显示的浮层，对应的层叠顺序越高。

    2
    3

    target="secondTarget"
    :options="{
      attachment: 'bottom left',
      targetAttachment: 'top left'
    }"

    overlay-class="relative-overlay"

  <veui-button

import { Overlay, Button } from 'veui'

    浮层之上弹出的浮层，对应的层叠顺序较高。

    浮层之上的浮层

    点击按钮，出现浮层，在浮层之上会再次看到一个按钮，点击该按钮，弹出子浮层，可发现子浮层始终位于父级浮层之上。

    <template>
<article>
  <veui-button
    ref="firstTarget"
    @click="firstVisible = !firstVisible"
  >{{ firstVisible ? '关闭' : '打开' }}基础浮层</veui-button>
  <veui-overlay
    target="firstTarget"
    :options="{
      attachment: 'bottom left',
      targetAttachment: 'top left'
    }"
    :open.sync="firstVisible"
    overlay-class="relative-overlay"
  >
    <veui-button
      ref="secondTarget"
      @click="secondVisible = !secondVisible"
    >{{ secondVisible ? '关闭' : '打开' }}浮层</veui-button>
    <veui-overlay
      target="secondTarget"
      :options="{
        attachment: 'bottom left',
        targetAttachment: 'top left'
      }"
      :open.sync="secondVisible"
      overlay-class="relative-overlay"
    >浮层之上的浮层</veui-overlay>
  </veui-overlay>
</article>
</template>

<script>
import { Overlay, Button } from 'veui'

export default {
  components: {
    'veui-overlay': Overlay,
    'veui-button': Button
  },
  data () {
    return {
      firstVisible: false,
      secondVisible: false
    }
  },
  watch: {
    firstVisible (val) {
      if (!val) {
        this.secondVisible = false
      }
    }
  }
}
</script>

    靠后显示的浮层应当遮盖之前显示浮层的所有子级浮层。

    A
    B
    B-A
    C

    第二个按钮对应的浮层里面有一个按钮，点击该按钮，会弹出子浮层。按照如下操作：

    1. 点击第二个按钮，弹出浮层 B；
    2. 点击浮层 B 上面的小按钮，弹出浮层 B-A；
    3. 点击第三个按钮，弹出浮层 C；
    4. 此时会发现浮层 C 覆盖于浮层 B-A 之上。

    只要保证 B 浮层在 C 浮层之前显示出来，那么 B-A 浮层就始终会位于 C 浮层之下。

    <template>
<article>
  <veui-button
    ref="firstTarget"
    @click="firstVisible = !firstVisible"
  >{{ firstVisible ? '关闭' : '打开' }}浮层</veui-button>
  <veui-overlay
    target="firstTarget"
    :options="{
      attachment: 'bottom left',
      targetAttachment: 'top left'
    }"
    :open.sync="firstVisible"
    overlay-class="relative-overlay"
  >A</veui-overlay>

  <veui-button
    ref="secondTarget"
    @click="secondVisible = !secondVisible"
  >{{ secondVisible ? '关闭' : '打开' }}浮层</veui-button>
  <veui-overlay
    target="secondTarget"
    :options="{
      attachment: 'bottom left',
      targetAttachment: 'top left'
    }"
    :open.sync="secondVisible"
    overlay-class="relative-overlay"
  >
    <span>B</span>
    <veui-button
      ui="small"
      ref="forthTarget"
      @click="forthVisible = !forthVisible"
    >{{ forthVisible ? '关闭' : '打开' }}浮层</veui-button>
    <veui-overlay
      target="forthTarget"
      :options="{
        attachment: 'bottom left',
        targetAttachment: 'top left'
      }"
      :open.sync="forthVisible"
      overlay-class="relative-overlay"
    >B-A</veui-overlay>
  </veui-overlay>

  <veui-button
    ref="thirdTarget"
    @click="thirdVisible = !thirdVisible"
  >{{ thirdVisible ? '关闭' : '打开' }}浮层</veui-button>
  <veui-overlay
    target="thirdTarget"
    :options="{
      attachment: 'bottom left',
      targetAttachment: 'top left'
    }"
    :open.sync="thirdVisible"
    overlay-class="relative-overlay"
  >C</veui-overlay>
</article>
</template>

<script>
import { Overlay, Button } from 'veui'

export default {
  components: {
    'veui-overlay': Overlay,
    'veui-button': Button
  },
  data () {
    return {
      firstVisible: false,
      secondVisible: false,
      thirdVisible: false,
      forthVisible: false
    }
  }
}
</script>

    API

    属性

    名称类型默认值描述
    uistring-浮层的主题。
    openbooleanfalse

    .sync

    是否显示对话框。

    targetstring|Vue|Node-

    允许通过 ref、Vue 组件实例、Node 的方式指定目标元素。如果目标元素存在，浮层会相对于该元素进行定位，具体定位参数由 options 属性指定。

    类型描述
    string在当前浮层组件所在上下文中，通过匹配 $refs 中的键名查找 DOM 节点或对应组件实例的根节点。
    Vue如果传入的是组件实例，就直接返回该组件的根节点。
    Node如果已经是一个 DOM 节点了，就直接使用该节点。
    optionsObject{}

    指定定位参数。仅当通过 target 成功找到 DOM 节点时才有效。具体设置，可参考 Tether 的文档 。

    prioritynumber-

    当前浮层组件实例与其它实例在层叠关系上的权重，数值越大，越靠上。

    由于所有浮层组件的浮层根元素都会放置到 <body> 元素下，所以为了更好地控制浮层层叠顺序，浮层模块内部实现了全局的层叠上下文。组件的父子关系决定了浮层的嵌套关系，在同层级内的每个浮层组件实例都可以通过指定 priority 来实现不同实例间的相对层叠顺序。相同 priority 数值的组件实例，按照实例化的先后顺序来决定层叠顺序，后实例化的组件在之前组件的上面。

    autofocusboolean-是否自动抢占焦点到浮层内的第一个可聚焦元素。
    modalbooleanfalse是否是模态对话框。模态对话框默认遮挡底层（无法点击）且抢占焦点（关闭后焦点会回归）。
    overlay-classstring|Array|Object-

    浮层根元素类名，数据格式为所有 Vue 支持的 class 表达式。

    由于浮层根元素会放置在 <body> 元素下，所以通过这个属性给浮层根元素设置类名之后，便于写样式。

    插槽

    名称描述
    default浮层内容。

    事件

    名称描述
    locate浮层定位发生变化时触发。
    afterclose浮层关闭后触发。如果样式主题提供了退出动画，将在退出动画完毕后触发。

    全局配置

    配置项类型默认值描述
    overlay.ovrerlayClassstring|Array|Object[]全局配置需要添加到浮层容器上的类名，数据格式为所有 Vue 支持的 class 表达式。
